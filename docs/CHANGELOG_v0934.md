# v0.9.34 — Detailed Bug, Event & Thai Wording Audit

## Critical fixes

- แก้เหตุการณ์เกิดเพิ่มประชากรซ้ำ ให้เพิ่มเด็กจริงเพียง 1 คน
- แก้สูตรงานเกินกำลังให้เทียบ “ภาระงานที่มอบหมาย” กับ “กำลังแรงงาน” ไม่ใช้ค่าผลผลิตคนละหน่วย
- แก้ Autosave ให้เก็บสถานะเกมครบและไม่เสี่ยงวนอัปเดต
- แก้คำสั่งต้มสมุนไพรให้ใช้ตามจำนวนผู้ป่วยจริงและไม่หักของเมื่อไม่มีผู้ป่วย
- แก้สถานะสาเหตุป่วย/บาดเจ็บค้างหลังรักษาหาย
- ปิดช่องโหว่รับผลประโยชน์จาก event เต็มจำนวนทั้งที่จ่ายทรัพยากรไม่ครบ
- แก้ ID ตัวเลือกเหตุการณ์ซ้ำทั่วทั้งคลัง event
- แก้ `package-lock.json` ไม่ให้ชี้ไปยัง registry ภายในที่เครื่องผู้ใช้เข้าถึงไม่ได้
- ปรับผู้มาตั้งถิ่นฐานประจำปีให้สัมพันธ์กับที่พัก อาหาร ระยะชุมชน และความน่าอยู่

## Event system

- เพิ่มตัวช่วยคำนวณค่าใช้จ่าย ของที่ขาด และสัดส่วนผลตอบแทน
- ผลบวกแบบแบ่งได้ลดตามสัดส่วนการชำระ
- ผลแบบแบ่งไม่ได้เกิดเฉพาะเมื่อชำระครบ
- ผลเสียของการตัดสินใจยังเกิดเต็มตามบริบท
- หน้าตัวเลือกแสดงค่าใช้จ่าย ของที่ขาด และผลที่ลดลงก่อนยืนยัน
- รายงานจบเดือนบันทึกผลจริงจาก event และผู้นำ

## Recovery and labor

- พักกลางแจ้งขั้นต่ำ 3 หน่วย/เดือน
- มีกองไฟขั้นต่ำ 8 หน่วย/เดือน
- ความแออัดไม่ลบค่าพื้นฐานจากกลางแจ้ง/กองไฟ
- ความเสี่ยงงานหนักใช้จำนวนคนและภาระงานจริง

## Wording/UI

- ปรับคำอังกฤษที่หลุดในภาษาไทยเป็นคำเดียวกันทั้งระบบ
- ปรับชื่อหมวด event และคำอธิบายเรื่องต่อเนื่อง
- ปรับข้อความ event ที่สร้างอัตโนมัติให้อ่านเป็นธรรมชาติ
- ใช้บทบาทตามบริบทแทน NPC ชื่อตายตัวในคำบรรยาย
- คงคำว่า JSON เฉพาะจุดนำเข้า/ส่งออกไฟล์
