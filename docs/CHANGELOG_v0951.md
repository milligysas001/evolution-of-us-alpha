# Evolution of Us v0.9.51 — Integrity, Recovery & Performance Patch

## ระบบบันทึกและกู้คืน

- เปลี่ยน Autosave, Manual Save และ Import ให้ใช้ Save Envelope รุ่นเดียวกัน
- ตรวจ Checksum ก่อนโหลดทุกช่องบันทึก
- เก็บข้อมูลรายงานจบเดือนใน Save ไม่ทิ้ง `summaryModal`
- เก็บ Autosave สำรองหมุนเวียน 5 จุด และลองกู้ตามลำดับเมื่อบันทึกล่าสุดเสีย
- เพิ่ม `saveRuntime` สำหรับจุดบันทึก เวลา และจำนวนครั้งที่กู้ข้อมูล
- จำกัดข้อมูลนำเข้าไม่เกิน 5 MB
- แจ้งผู้เล่นเมื่อพื้นที่จัดเก็บเบราว์เซอร์เต็มหรือบันทึกอัตโนมัติไม่สำเร็จ

## Monthly Flow และ Ledger

- สถานะเดือนถูกคำนวณจากการเลือกจริง: Planning → Decision → Ready → Resolving → Report
- ป้องกันการคำนวณเดือนเดิมซ้ำด้วย Resolution ID และประวัติเดือนที่คำนวณแล้ว
- โหลดระหว่าง Resolving จะกลับสู่ Ready อย่างปลอดภัย
- โหลดระหว่าง Report จะเปิดรายงานเดิมก่อนเข้าสู่เดือนถัดไป
- การเปลี่ยนทรัพยากรระหว่างเดือนถูกบันทึกเป็น State Diff Ledger
- เก็บ Ledger เดือนปัจจุบันสูงสุด 500 รายการ และประวัติย้อนหลัง 48 เดือน

## Event Integrity

- ตรวจ Runtime จริง 338 เหตุการณ์ 1,017 ตัวเลือก
- ตรวจ ID ซ้ำ ทางเลือกว่าง ผลลัพธ์ว่าง รหัสทรัพยากรผิด และ Event ต่อเนื่องหาย
- ตรวจข้อความทางเลือกที่กว้างเกินไปหรือบอกว่ามีต้นทุนแต่ไม่หักจริง
- แก้ทางเลือก “ยังไม่มีเวลาทดลอง” ให้มีผลด้านสมาธิและการอยู่รอดจริง
- ผลตรวจ Runtime ไม่เหลือ Error หรือ Warning

## ประสิทธิภาพและ UX

- รายชื่อคนค้นหาตามชื่อ บทบาท และคุณลักษณะได้
- เรียงตามความเร่งด่วน ชื่อ ความล้า หรือสุขภาพ
- แบ่งหน้า 40 คน ลดการ Render รายชื่อทั้งหมดพร้อมกัน
- Autosave ใช้ Debounce 1.6 วินาทีและบันทึกจุดสำคัญเท่านั้นสำหรับ Backup/Leader Board
- นำรหัสผู้พัฒนาที่ฝังใน Client ออก เครื่องมือภายในแสดงเฉพาะ Development Build
- ผู้เล่นนำเข้าบันทึกได้จากหน้าระบบบันทึกโดยไม่ต้องเปิดเครื่องมือนักพัฒนา
