# Quality Check — v0.9.6

ตรวจและแก้ไขแล้ว:

- ตรวจ TSX syntax ด้วย TypeScript transpile diagnostics: ผ่าน ไม่มี syntax error
- แก้ React duplicate key warning จาก tag ภาษาไทยที่ซ้ำกัน โดยเปลี่ยน key เป็น id + text + index
- ตรวจ flow หลัก:
  1. ตั้งชื่อ / เลือกพื้นหลัง
  2. เข้าเกม
  3. จัดแรงงาน
  4. เลือกก่อสร้าง / วิจัย
  5. เลือกบทบาทผู้นำ
  6. เลือก event choice
  7. จบเดือน
  8. คำนวณผลผลิต การบริโภค ความเสี่ยง บาดเจ็บ/เสียชีวิต
  9. ตรวจเงื่อนไขล่มสลาย
  10. ถ้าแพ้ แสดง Game Over พร้อมปุ่มเริ่มใหม่

ระบบแพ้ที่เพิ่ม:

- ประชากรเหลือ 0: แพ้ทันที
- อดอาหารต่อเนื่อง 4 เดือน: แพ้จากความอดอยาก
- ความไว้ใจ <= 8 และสามัคคี <= 18 ต่อเนื่อง 3 เดือน: ผู้นำถูกขับออก
- ภัยภายนอก >= 92 และความปลอดภัย <= 18 ต่อเนื่อง 2 เดือน: ค่ายแตก
- ไม่มีแรงงาน 4 เดือนและอาหารไม่พอ: แพ้จากไร้แรงงาน

UX ที่แก้:

- รายชื่อผู้ตายแสดงแบบย่อ 5 คนล่าสุด
- รายชื่อทั้งหมดเปิดด้วย details ไม่ดันกรอบยาวลงเรื่อย ๆ
- เพิ่มระดับวิกฤตใน topbar และ profile
- ปุ่มจบเดือนจะไม่ทำงานซ้ำหลัง Game Over
