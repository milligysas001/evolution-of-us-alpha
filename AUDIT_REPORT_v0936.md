# รายงานตรวจระบบ Evolution of Us v0.9.36

## 1. ระบบบันทึก

ระบบเดิมมี Autosave ชุดเดียวใน `localStorage` ซึ่งเสี่ยงต่อการบันทึกทับสถานะที่ผู้เล่นอยากย้อนกลับ เวอร์ชันนี้เพิ่มโครงสร้างดังนี้

- `eou-current-save` เป็น Autosave ปัจจุบัน
- `eou-autosave-backup-v1` เก็บสถานะก่อนหน้าทุกครั้งที่ข้ามเดือน
- `eou-save-slots-v1` เก็บบันทึกด้วยตนเองสูงสุด 3 ช่อง
- `eou-active-save-slot-v1` ระบุช่องที่กำลังเล่น เพื่อ Autosave กลับเข้าช่องนั้น
- ดาวน์โหลด JSON เพื่อย้ายอุปกรณ์หรือเก็บสำรองนอกเบราว์เซอร์

แก้บั๊กหน้าแรกที่ตรวจ `window.localStorage` ระหว่าง Server Render แล้วค่า “เล่นต่อ” ไม่อัปเดต โดยย้ายการตรวจไปทำหลัง Hydration ผ่าน `useEffect`

## 2. Leader Board

อันดับคำนวณโดยให้น้ำหนักตามลำดับ:

1. ยุค/ขนาดเมือง
2. ปีและเดือนที่อยู่รอด
3. จำนวนประชากร
4. ค่าเฉลี่ยสภาพเมือง

แต่ละคู่ `ชื่อตระกูล + ชื่อผู้นำ` เก็บเฉพาะคะแนนสูงสุด เพื่อให้การเริ่มรอบใหม่ที่ทำได้แย่กว่าไม่ลบสถิติเดิม

**ข้อจำกัด:** เป็น Leader Board บนอุปกรณ์เดียวกัน ยังไม่ใช่อันดับออนไลน์ เนื่องจากไม่มีฐานข้อมูลกลางหรือระบบบัญชีผู้เล่น

## 3. แยกข่าว พ่อค้า และการค้า

- ข่าวสารแสดงข่าวลือ ภัย และสัญญาณจากเมืองอื่นเท่านั้น
- พ่อค้าเร่แสดงเมื่อ Event ปัจจุบันมีหมวด/ชื่อเกี่ยวกับพ่อค้าหรือคาราวาน และระบบการค้าถาวรยังไม่เปิด
- การค้าถาวรเปิดเมื่อมีการผลิตเหรียญ สัญญาคาราวาน ลานตลาด สถานีคาราวาน หรือเข้าสู่เมืองการค้า
- เมื่อเปิดการค้าถาวร แท็บพ่อค้าเร่จะไม่แสดงอีก จึงไม่มีหน้าซ้ำซ้อน

## 4. เมืองข้างเคียง

เพิ่มเมืองต้นแบบ 5 เมือง โดยแต่ละเมืองมีค่าที่ใช้จริง:

- ประชากรและขนาดเขตแดน
- ความสัมพันธ์ ความไว้ใจ ความกลัว และความตึงเครียด
- ความชำนาญทรัพยากร
- สนธิสัญญาการค้า พันธมิตร และสงคราม

Event `neighbor_smoke_beyond_ridge` เปิดระบบครั้งแรกหลังถึงหมู่บ้านถาวร ส่วน `neighbor_border_dispute` ถูกเพิ่มเข้า Pending Event เมื่อความตึงเครียดชายแดนสูง

การกระทำแต่ละแบบตรวจต้นทุนและเงื่อนไขจริง สนธิสัญญาสร้างผลผลิตรายเดือน ส่วนสงครามเปรียบเทียบพลังรบของผู้เล่นกับขนาดประชากร เขตแดน และความกลัวของเมืองคู่สงคราม

## 5. การทหาร

ระบบไม่ปรากฏในค่ายพักแรมและชุมชนแรกเริ่ม ต้องถึงหมู่บ้านถาวรและวิจัย `militiaTraining`

เพิ่ม:

- งานวิจัย 3 รายการ
- อาคาร 2 รายการ
- กำลังพล ความพร้อม ขวัญ อุปกรณ์ ประสบการณ์ และท่าที
- ค่าอาหารและทองรายเดือน
- การฝึก การจัดอุปกรณ์ การปลดประจำการ
- พลังรบสำหรับเหตุการณ์ชายแดนและสงคราม

หากเสบียงทหารไม่พอ ความพร้อมและขวัญจะลดลงจริง

## 6. การจัดแรงงาน

ลดความสูงขั้นต่ำจาก 68 เป็น 54 พิกเซลบนจอใหญ่ ลด padding, badge และ select แต่ยังแสดง:

- ชื่อ บทบาท อายุ และกำลังแรงงาน
- คุณลักษณะสำคัญ 2 รายการ
- ป่วย บาดเจ็บ ล้าสูง หรือพร้อมทำงาน
- งานปัจจุบัน/ค่าพักฟื้น
- สุขภาพ ความล้า และช่องเลือกงาน

บนแท็บเล็ตเปลี่ยนเป็นโครง 2 คอลัมน์แทนการแตกเป็นหลายบรรทัดทันที และจะแตกเป็นคอลัมน์เดียวเฉพาะจอไม่เกิน 760 พิกเซล

## 7. ผลตรวจถดถอย

- สุ่มสร้างเกม 250 รอบ: ประชากร 15 ทุกครั้ง
- สิ่งก่อสร้าง 31 รายการ: มีต้นทุนมากกว่า 0 ทุกทรัพยากร
- งานวิจัย 41 รายการ: Stage requirement ถูกต้อง
- Event 338 รายการ / Choice 1,017 ตัวเลือก: ID ไม่ซ้ำและประมวลผลโดยไม่เกิด `NaN`
- พ่อค้าเร่แสดงก่อนการค้าถาวรและหายหลังปลดล็อกถาวร
- เมืองข้างเคียงถูกเพิ่มจาก Event และสนธิสัญญาสร้างทรัพยากรรายเดือน
- ระบบทหารเปิดตามยุคและวิจัย ค่าเลี้ยงดูลดอาหารจริง
- Production Build และ HTTP ผ่าน
